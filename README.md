### Brief Summary:
 - I have built several RAG flows, and for this assessment I used one based on a more object-oriented approach with `LangGraph`, which I recently learned.
 - The data is synthetic, generated by an LLM. The chunking strategy is `chunk_size/chunk_overlap` → `700/100`. The longest text is no more than 700 characters, and the overlap may capture only whitespace since paragraphs are separated by at least one empty line. That said, it might be better to reduce the overlap.
 - I also added a ReAct agent that searches for relevant information on a specific website when the data cannot be retrieved from the text files.
 - One of the key decisions I made was choosing the models for generation and embeddings. The free-tier models struggled to generate satisfactory outputs, so I am using the paid-tier `gemini-2.5-flash`. I also switched from ChromaDB to in-memory FAISS, which is faster and well-suited for testing.


### Project Setup and how to execute it locally.
 1. In the `.env` file you `must` specify your `GOOGLE_API_KEY` provided by: `https://aistudio.google.com/welcome` (free trial is available)
 2. I used `uv` for package management and im providing `pyproject.toml` file, along with `requirements.txt` if needed.
 3. For UI im using `streamlit`. To run the application, go to the terminal and execute the following command: `streamlit run streamlit_app.py` If the default browser does not open automatically, check the terminal for the localhost address where the application is running.
